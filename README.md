# GeoCollect Field App Platform

Open-architecture geospatial data collection platform. Alternative to ArcGIS Survey123 & FieldMaps.

## Products

| Product | Directory | Description |
|---------|-----------|-------------|
| **Field Mobile App** | `mobile/` | React Native (Expo) offline-first data collection app |
| **Form Builder (Studio)** | `studio/` | Next.js form designer & project management portal |
| **API** | `api/` | Fastify REST API (auth, forms, submissions, orgs, mobile) |
| **Geo API** | `geo-api/` | FastAPI geospatial service (export, validation) |
| **Field PWA** | `field/` | React PWA (web-based collection, legacy) |
| **Portal** | `portal/` | Next.js analytics portal |

## Architecture

```
Mobile App (React Native)
    ↕ HTTPS
Fastify API (Render) ←→ Neon Postgres (PostGIS)
    ↕                ←→ Redis (Render)
FastAPI Geo API (Render)
    ↕
Studio / Portal (Render)
```

## Deployment — Render + Neon

### 1. Create a Neon database

1. Sign up at [neon.tech](https://neon.tech) (free tier available)
2. Create a new project → database name: `geocollect`
3. Enable PostGIS: in the Neon SQL editor run:
   ```sql
   CREATE EXTENSION IF NOT EXISTS postgis;
   CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
   ```
4. Copy the **Connection string** (format: `postgresql://user:pass@ep-xxx.neon.tech/geocollect?sslmode=require`)

### 2. Deploy to Render

```bash
# Connect your repo to Render
# Then apply the blueprint:
render blueprint apply render.yaml
```

Or manually:
1. Go to [render.com](https://render.com) → New → Blueprint
2. Connect this repo
3. Render reads `render.yaml` and creates all services automatically

### 3. Set the DATABASE_URL

In the Render dashboard, for **both** `geocollect-api` and `geocollect-geo-api`:
- Go to Environment → Add environment variable
- Key: `DATABASE_URL`
- Value: your Neon connection string (from step 1)

### 4. Migrations run automatically

The API's start command runs `node src/db/migrate.js` before starting, which creates all tables including the new org/mobile tables (v10).

### 5. Build the mobile APK

```bash
cd mobile
npm install
cp .env.example .env
# Edit .env: set EXPO_PUBLIC_API_URL=https://geocollect-api.onrender.com
eas login
eas build --platform android --profile preview
```

Download the APK link from the terminal or [expo.dev](https://expo.dev).

## Local Development

```bash
# Start all services
docker compose up --build

# Or individual services:
cd api && npm install && npm run dev
cd geo-api && pip install -r requirements.txt && uvicorn app.main:app --reload
cd studio && npm install && npm run dev
cd mobile && npm install && npx expo start
```

## Environment Variables

See `.env.example` files in each service directory.

**Required for production:**
- `DATABASE_URL` — Neon connection string (set in Render dashboard)
- `JWT_SECRET` — Auto-generated by Render
- `REDIS_URL` — Auto-injected by Render from the Redis service

**Optional:**
- `SMTP_*` — For invite emails (use Resend, SendGrid, etc.)
- `MINIO_*` — For photo/attachment storage (use Cloudflare R2 or AWS S3)

## Mobile App Features

- ✅ Offline-first data collection (SQLite)
- ✅ Dynamic form renderer (13 field types)
- ✅ GPS capture + track recording
- ✅ Geofencing (polygon/radius, block/warn/track modes)
- ✅ Photo capture with EXIF GPS
- ✅ Barcode/QR scanning
- ✅ Multi-format export: CSV, GeoJSON, KML, Shapefile, DXF, JSON
- ✅ Organization accounts with role-based access
- ✅ Email invite flow (Admin → Member)
- ✅ Background sync when connection restored
- ✅ APK buildable via Expo EAS

## Roles

| Role | Form Builder | Mobile App |
|------|-------------|------------|
| Org Admin | Full access | Full access |
| Form Editor | Create & edit forms | Collect data |
| Data Collector | View assigned forms | Collect data only |
| Viewer | Read-only | No access |
